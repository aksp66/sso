Authentification OAuth2 + PKCE

Nexus implémente le flux Authorization Code (RFC 6749) avec extension PKCE (RFC 7636).

Pourquoi PKCE ?

PKCE protège le flux contre l'interception du code d'autorisation, notamment sur les clients qui ne peuvent pas garder de secret (SPA, mobile, CLI). Nexus l'exige pour tout client enregistré comme public (is_confidential = false).

Génération du code_verifier / code_challenge

// JavaScript
const codeVerifier = base64url(crypto.getRandomValues(new Uint8Array(32)));
const codeChallenge = base64url(await sha256(codeVerifier));
// code_challenge_method = "S256"
# Python
import secrets, hashlib, base64
code_verifier = secrets.token_urlsafe(32)
code_challenge = base64.urlsafe_b64encode(
    hashlib.sha256(code_verifier.encode()).digest()
).decode().rstrip("=")
code_verifier doit contenir entre 43 et 128 caractères (RFC 7636 §4.1). Nexus rejette toute valeur hors de cette plage avec invalid_grant.

Le paramètre state

Toujours envoyer un state aléatoire et unique par requête, le stocker côté client (session ou cookie), puis vérifier qu'il correspond exactement à la valeur retournée par Nexus après redirection. Cela protège votre application contre les attaques CSRF sur le flux d'autorisation (RFC 6749 §10.12).

Rotation des refresh tokens

Chaque utilisation d'un refresh_token révoque l'ancien et émet un nouveau (RFC 6749 §10.4). Ne réutilisez jamais un refresh token déjà échangé — il sera rejeté avec invalid_grant.
