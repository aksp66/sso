Démarrage rapide

Intégrez l'authentification Nexus à votre application en quelques étapes.

1. Enregistrer votre application

Un administrateur Nexus doit enregistrer votre application comme client OAuth2 depuis le panneau d'administration (/admin/clients/new). Vous recevrez :

  • client_id — identifiant public de votre application
  • client_secret — secret confidentiel (uniquement pour les applications serveur)
Si votre application est une SPA, une app mobile ou tout autre client qui ne peut pas garder un secret confidentiel, elle doit être enregistrée comme client public. PKCE est alors obligatoire.

2. Rediriger l'utilisateur vers Nexus

GET /authorize?
  response_type=code
  &client_id=VOTRE_CLIENT_ID
  &redirect_uri=https://votre-app.com/callback
  &scope=openid email profile
  &state=VALEUR_ALEATOIRE
  &code_challenge=XXXX        (clients publics)
  &code_challenge_method=S256

L'utilisateur se connecte, voit l'écran de consentement, puis est redirigé vers votre redirect_uri avec un code.

3. Échanger le code contre des tokens

curl -X POST https://sso.example.com/token \
  -u "VOTRE_CLIENT_ID:VOTRE_CLIENT_SECRET" \
  -d grant_type=authorization_code \
  -d code=LE_CODE_RECU \
  -d redirect_uri=https://votre-app.com/callback \
  -d code_verifier=VALEUR_GENEREE_A_L_ETAPE_2

Réponse :

{
  "access_token": "eyJhbGciOi...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "refresh_token": "AbCdEf...",
  "id_token": "eyJhbGciOi...",
  "scope": "openid email profile"
}

4. Récupérer les informations utilisateur

curl https://sso.example.com/userinfo \
  -H "Authorization: Bearer eyJhbGciOi..."

Pour la suite, consultez la page Authentification pour comprendre PKCE en détail, ou la référence API complète.
