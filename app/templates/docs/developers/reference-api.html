Référence API

Liste complète des endpoints exposés par Nexus.

GET /.well-known/openid-configuration

Métadonnées de découverte OpenID Connect.

GET /jwks.json

Clés publiques RS256 actives, au format JWK Set. Utilisé pour vérifier la signature des tokens.

GET /authorize

ParamètreRequisDescription
response_typeOuiDoit être code
client_idOuiIdentifiant du client
redirect_uriOuiURI enregistrée pour ce client
scopeOuiopenid, email, profile
stateRecommandéAnti-CSRF, renvoyé à l'identique
code_challengePublic : ouiPKCE — SHA-256 du verifier
code_challenge_methodSi PKCEDoit être S256
nonceOptionnelAnti-replay, inclus dans l'ID token

POST /token

Authentification client via Authorization: Basic ou champs POST client_id/client_secret.

grant_type=authorization_code

grant_type=authorization_code
&code=...
&redirect_uri=...
&code_verifier=...   # si PKCE

grant_type=refresh_token

grant_type=refresh_token
&refresh_token=...

Réponse 200

{
  "access_token": "",
  "token_type": "Bearer",
  "expires_in": 3600,
  "refresh_token": "",
  "scope": "openid email",
  "id_token": ""
}

GET /userinfo

En-tête Authorization: Bearer <access_token>. Les claims retournés dépendent du scope du token — voir Scopes & claims.

POST /revoke

Révocation RFC 7009. Retourne toujours 200.

token=...
&token_type_hint=access_token|refresh_token

GET /health

État des composants (base de données, Redis, clé RS256 active). Utilisable comme probe de santé.
